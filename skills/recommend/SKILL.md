사용자의 현재 작업 맥락을 분석하여 설치된 도구 중 가장 적합한 것을 추천한다.

## Step 1: 인벤토리 읽기

Bash 도구로 `echo $HOME`을 실행하여 홈 디렉토리 절대 경로를 확인한다. `~` 기호는 Read 도구의 file_path에 사용할 수 없으므로 반드시 절대 경로를 사용한다.

확인된 경로를 기반으로 Read 도구로 다음 파일을 읽는다.

경로: `{HOME}/.claude/cache/claude-radar/inventory.json`

파일 읽기에 실패하거나 파일이 존재하지 않으면 다음 안내를 출력하고 종료한다.

```
도구 인벤토리가 없습니다. 새 Claude Code 세션을 시작하면 자동으로 스캔됩니다.
```

## Step 2: 맥락 파악

사용자의 현재 요청 및 대화 맥락을 분석하여 어떤 작업을 하려는지 파악한다.

맥락이 명확하게 파악되면 Step 3으로 바로 이동한다.

맥락이 불명확하면 다음 한 가지 질문만 하고 사용자의 답변을 기다린다.

```
어떤 작업을 하려고 하세요? (목적, 사용 언어/프레임워크, 현재 단계를 알려주시면 더 정확하게 추천해드릴 수 있습니다.)
```

## Step 3: 도구 매칭

인벤토리의 `tools` 배열 전체를 검토한다. 인벤토리 구조에 따라 최상위 키가 다를 수 있으므로, 읽은 JSON의 실제 구조를 기반으로 도구 목록을 추출한다. `tools` 배열이 없다면 `mcpServers`, `plugins`, `skills`, `agents`, `commands` 등 개별 섹션을 합산하여 전체 도구 목록을 구성한다.

각 도구의 `description`, `keywords`, `categories` 필드를 사용자 맥락과 비교하여 LLM의 자연어 이해를 바탕으로 적합도를 판단한다. 단순 키워드 매칭이 아니라 작업의 본질과 도구의 목적이 얼마나 부합하는지를 기준으로 삼는다.

적합도 기준:
- 높음: 도구가 현재 작업과 직접적으로 관련
- 중간: 현재 작업에 부가적으로 유용
- 낮음: 간접적으로 관련

`enabled: false`인 도구는 추천 대상에 포함하되 낮은 순위로 배치하고 이름 옆에 "(비활성)" 표시를 추가한다.

적합도 높음 → 중간 → 낮음 순으로 정렬하여 상위 3~5개를 선정한다. 관련 도구가 5개를 초과하면 가장 직접적으로 관련된 것만 선택한다.

## Step 4: 결과 표시

다음 형식으로 출력한다. `{task_context}`는 파악한 작업 맥락을 2~5단어로 요약한 것이다.

```
## 추천 도구 ({task_context} 작업에 적합)

| 순위 | 도구 | 유형 | 적합도 | 왜 적합한가 |
|------|------|------|--------|-------------|
| 1 | {name} | {type} | {적합도} | {이유} |
| 2 | {name} | {type} | {적합도} | {이유} |
| 3 | {name} | {type} | {적합도} | {이유} |

**호출 방법**:
- {name}: {invocation 필드 값 또는 유형별 호출 방법}
- {name}: {invocation 필드 값 또는 유형별 호출 방법}
- {name}: {invocation 필드 값 또는 유형별 호출 방법}
```

`invocation` 필드가 있으면 해당 값을 그대로 사용한다. 없을 경우 유형별 기본 호출 방법을 안내한다.
- mcp: `mcp__{name}__*` 형태의 MCP 도구 사용
- skill: Skill 도구로 `{name}` 호출
- agent: Agent 도구로 `{name}` 실행
- plugin-skill: Skill 도구로 `{plugin}:{name}` 호출
- plugin-agent: Agent 도구로 `{name}` 실행
- command: `/command {name}` 실행
- plugin: 플러그인 내 도구를 통해 사용

## Step 5: 실행 여부 확인

결과 표시 후 다음 문장을 출력한다.

```
추천된 도구 중 지금 바로 실행할 것이 있나요? 도구 이름을 말씀해주시면 실행 방법을 안내해드립니다.
```

## 주의사항

- Read 도구의 file_path에는 반드시 절대 경로를 사용한다 (`~` 사용 불가).
- 추천 개수는 3~5개로 제한한다. 적합한 도구가 3개 미만이면 있는 것만 표시한다.
- 모든 도구 유형(mcp, skill, agent, plugin, plugin-skill, plugin-agent, command)을 동등하게 고려한다.
- 이유 설명은 간결하게 한 문장으로 작성한다.
- 유형 표시는 원본 `type` 필드 값을 그대로 사용하되 가독성을 위해 대문자로 변환한다 (예: `mcp` → `MCP`, `skill` → `Skill`).
